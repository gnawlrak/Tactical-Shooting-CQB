# 游击战术系统 - AI 防守增强

## 🎯 核心设计理念

**用最小的伤亡换取最大的伤害** - 敌人不再是无脑冲锋的靶子，而是会利用地形、掩体进行游击战的智能对手。

---

## ✨ 新增功能概览

### 1. **动态暴露风险评估系统**

AI 会实时评估当前位置的危险程度：

```gdscript
风险值 = 距离风险 + 开阔地带风险 + 暴露时间风险 + 交叉火力加成
```

| 风险因素 | 评分标准 |
|---------|---------|
| **距离玩家<15m** | +30 分（非常危险） |
| **距离玩家 15-30m** | +15 分（中等危险） |
| **不在掩体附近** | +40 分（极度危险） |
| **连续暴露时间** | ×2 分/秒 |
| **有 2+ 队友看着** | -20 分（相对安全） |

**高风险行为**（>50 分）:
- 主动转移到更安全的位置
- 降低姿态（蹲下或匍匐）
- 减少射击频率

---

### 2. **游击战术位置选择**

#### 2.1 埋伏位置选择（IDLE 状态）
```gdscript
理想埋伏点 = 
    ✓ 距离玩家 20-40m（中等距离，既有威胁又安全）
    ✓ 周围 5m 内有其他掩体（便于转移）
    ✓ 没有其他敌人占据
    ✓ 有良好的射界（能打到玩家但玩家打不到）
```

#### 2.2 推进掩体选择（CHASE 状态）
```gdscript
推进掩体 = 
    ✓ 比当前位置更靠近玩家
    ✓ 容易到达
    ✓ 可以作为下一步进攻的跳板
```

#### 2.3 撤退掩体选择（RETREAT 状态）
```gdscript
撤退掩体 = 
    ✓ 在远离玩家的方向上
    ✓ 能提供良好保护
    ✓ 有多个撤退路线
```

---

### 3. **战术行为改进**

#### 3.1 防守状态（DEFEND）- 显著增强

**旧行为**:
- 蹲在原地不动
- 计时器到了就冲锋
- 玩家靠近才反应

**新行为**:
```gdscript
✓ 持续评估暴露风险
✓ 风险>50 时主动转移（3-6 秒重新评估）
✓ 优先寻找掩体
✓ 采用蹲姿或匍匐姿
✓ 短点射（2-4 发）而非长点射
✓ 射击间隔延长（1.5-3.0 秒 → 2-4 秒）
✓ 射击后缩回掩体
✓ 80% 概率继续防守，不轻易离开掩体
```

**典型场景**:
> 玩家在远处看到敌人蹲在掩体后，刚准备瞄准，敌人已经缩回掩体并转移到另一个位置。

---

#### 3.2 追击状态（CHASE）- 利用掩体推进

**旧行为**:
- 直线冲向玩家
- 边跑边开枪
- 完全暴露在火力下

**新行为**:
```gdscript
✓ 寻找推进掩体（比当前位置更靠近玩家的掩体）
✓ 低姿态前进（蹲姿）
✓ 只在有掩体保护时射击
✓ 侧翼移动保持不变
```

**典型场景**:
> 敌人不是直线冲过来，而是利用地图上的箱子、墙壁作为掩护，一步步逼近玩家。

---

#### 3.3 攻击状态（ATTACK）- 游击战核心

**旧行为**:
- 站在原地对射
- 长时间暴露
- 连续射击不隐蔽

**新行为**:
```gdscript
✓ 持续评估暴露风险（>60 立即转移）
✓ 优先寻找掩体
✓ 在掩体间移动（蹲姿）
✓ 短点射（2-4 发）
✓ 射击间隔 2-4 秒
✓ 射击后缩回掩体
✓ 自动朝掩体方向移动
```

**典型场景**:
> 敌人从掩体后探身射击 2-3 发子弹，然后立即缩回，换个位置继续射击。

---

#### 3.4 撤退状态（RETREAT）- 战术转移

**旧行为**:
- 站立逃跑（目标大）
- 不利用掩体

**新行为**:
```gdscript
✓ 蹲姿撤退（减小被命中面积）
✓ 朝向掩体撤退
✓ 保持面向玩家方向（随时准备反击）
```

---

### 4. **隐蔽模式**

新增变量：`is_in_stealth_mode`

**触发条件**:
- 移动到埋伏点时
- 执行战术转移时
- 暴露风险过高时

**效果**:
- 自动采用蹲姿
- 移动速度降低
- 减少射击频率
- 优先寻找掩体

## 📊 数值对比

### 射击行为对比

| 指标 | 旧版 | 新版 | 改进 |
|------|------|------|------|
| **每轮射击次数** | 3-7 发 | 2-4 发 | ↓ 减少暴露时间 |
| **射击间隔** | 0.8-1.5 秒 | 2.0-4.0 秒 | ↑ 提高生存性 |
| **防守转移概率** | 100% | 20% | ↓ 更保守 |
| **射击后行为** | 原地不动 | 缩回掩体 | ↑ 提高生存性 |

### 姿态使用对比

| 状态 | 旧版 | 新版 |
|------|------|------|
| **DEFEND** | 蹲姿 | 蹲姿/匍匐 |
| **CHASE** | 站姿 | 蹲姿（利用掩体） |
| **ATTACK** | 随机 | 优先蹲姿 + 掩体 |
| **RETREAT** | 站姿 | 蹲姿 |

## 🎮 实际战斗场景

### 场景 1：远距离遭遇
```
旧版：敌人站在原地对射 → 被玩家逐个击破
新版：敌人利用掩体，短点射后转移 → 玩家难以锁定目标
```

### 场景 2：近距离突袭
```
旧版：敌人无脑冲锋 → 被玩家轻松收割
新版：敌人利用地形掩护，逐步逼近 → 玩家压力大增
```

### 场景 3：多人协作
```
旧版：所有敌人同时冲锋 → 容易被一网打尽
新版：部分敌人吸引火力，其他敌人侧翼包抄 → 玩家顾此失彼
```

## 🔧 技术实现

### 新增变量（11 个）

```gdscript
var preferred_cover_positions : Array       # 偏好掩体位置列表
var current_tactical_goal : String          # 当前战术目标
var exposure_time : float                   # 暴露时间
var concealment_bonus : float               # 隐蔽加成
var reposition_timer : float                # 转移计时器
var ambush_spot : Vector3                   # 埋伏点
var is_in_stealth_mode : bool               # 隐蔽模式
```

### 新增函数（9 个）

1. `initialize_tactical_positions()` - 初始化战术位置
2. `evaluate_exposure_risk()` - 评估暴露风险
3. `is_near_cover()` - 检查是否在掩体附近
4. `count_allies_with_los()` - 统计有视线的队友
5. `select_guerrilla_position()` - 选择游击位置
6. `count_nearby_covers()` - 统计附近掩体数量
7. `count_enemies_near_position()` - 统计某位置附近的敌人
8. `perform_tactical_relocation()` - 执行战术转移
9. `select_ambush_position()` - 选择埋伏点
10. `has_good_firing_angle()` - 检查射界
11. `update_ambush_position()` - 更新埋伏位置
12. `find_advance_cover()` - 寻找推进掩体

### 修改函数（4 个）

1. `process_idle()` - 添加埋伏逻辑
2. `process_defend()` - 添加风险转移逻辑
3. `process_chase()` - 添加掩体推进逻辑
4. `process_attack()` - 添加游击战术逻辑

## 📈 预期效果

### 游戏体验提升
1. **更有挑战性** - 敌人不再是活靶子，需要认真对待
2. **更真实** - 像真正的士兵一样战斗
3. **更需要策略** - 玩家需要思考如何对付游击战术
4. **更少挫败感** - 敌人变强是因为智能，不是因为数值

### 性能影响
- **轻微增加** - 增加了风险评估和位置选择的计算
- **优化措施** - 使用缓存和定时器，避免每帧计算
- **推荐配置** - 同屏敌人不超过 20 个

## 💡 设计哲学

> **"静止不动是无意义的等死"**

这是本次优化的核心理念。在真实的战斗中，士兵不会站在原地等待被击中，而是会：
1. **不断移动** - 让敌人难以瞄准
2. **利用掩体** - 减少被命中的概率
3. **短促交火** - 射击后立即隐蔽
4. **战术转移** - 发现危险立即换位置

现在的 AI 真正实现了这些行为，让玩家感受到与智能对手的较量，而不是简单的打靶练习。
